Skills（スキル）

AIに「特定の作業のやり方」を覚えさせておくファイル一式。Markdownの指示書に、必要ならスクリプトや参考資料を添えて1つのフォルダにまとめる。

プロンプトとの違い

プロンプトは会話のたびに書く単発の指示。Skillsは一度作れば、Claudeが必要なタイミングで自動的に読み込んで使う。毎回同じ説明を書く手間がなくなる。

仕組みの要点

  • 最初から全部読み込むのではなく、必要になったときだけ詳細を読み込む（段階的開示）
  • Claude.ai、Claude Code、APIなど環境をまたいで同じフォルダが使える
  • ChatGPTにも「カスタムGPT」やプロジェクト機能という近い仕組みがある
参考：Anthropic公式ドキュメント「Agent Skills」

MCP（Model Context Protocol）

AIが外部のサービスやデータに接続するための、共通の規格。Google Drive、Slack、GitHubなどと繋ぐ「共通コンセント」にあたる。

AI（Claude / ChatGPT）
⇄
MCPサーバー
⇄
外部サービス

Skillsとの役割分担

MCPが「何にアクセスできるか」を提供し、Skillsが「それをどう処理するか」を定義する。たとえばMCPでGitHubの差分を取得し、Skillsに書いたチェックリストでレビューする、という組み合わせ方が一般的。

  • ChatGPTでは「コネクタ」「Apps」という名称で同じ仕組みが提供されている
  • 対応しているAIアプリはClaude、ChatGPT、Cursor、VS Codeなど多数
参考：modelcontextprotocol.io、Anthropic／OpenAI公式ドキュメント

ハーネス

AIを安定して実務で使うための「実行環境＋安全装置」。Claude CodeやCodex CLIのような、ターミナルやIDEで動くエージェントツール全体を指す言葉。

Claude Code

細かい制御が可能なフック機構で安全性を確保。長期のリファクタリングやコードレビューに強い。

Codex CLI

OSレベルのサンドボックスで安全性を確保。境界が固く、クラウド実行と相性がよい。

共通点

どちらもSkills相当の拡張ファイルを持ち、MCPで外部サービスに繋がる構造は同じ。

設計思想は異なるが、性能差より「運用設計の差」が結果を左右するという指摘が多い（2026年時点の業界記事より）

プロンプトの基本原則

曖昧な指示は曖昧な答えを返す。Skills・MCP・ハーネスより前に効く、最も投資対効果が高い基本動作。

役割を指定する
「編集者として」「初心者向けに」など、立場を渡すと答えの質が揃う
条件を渡す
対象者・前提知識・避けたい表現などを先に伝える
出力形式を指定する
表で／箇条書きで／何文字以内で、と形を決める
ステップで考えさせる
複雑な作業は「まず分解して」と一段挟むと精度が上がる

チャットの構造化機能

Skillsの「個人用・簡易版」にあたる機能。毎回同じ前提を書き直さなくて済む。

Claude

Projects機能で、特定の話題用の会話スペースとカスタム指示をまとめて持てる

ChatGPT

Custom InstructionsとProjects機能で同様のことができる

Skillsとの違い

Skillsはファイル単位で配布・共有できる。構造化機能は個人の会話設定に近い

メモリ機能

会話を超えて、好みや過去のやり取りを覚えておく機能。

Skillsとの違い

Skillsは「作業のやり方」を教える仕組み。メモリは「あなた個人の情報」を覚える仕組み。役割が違うので混同しないことが大事。

「調べて」「実行して」「作って」、それぞれ投げる先の機能が違う。

調べてほしいとき

実行してほしいとき
Agent mode・Computer use系の機能
作ってほしいとき
Skills・コード実行機能
繋いでほしいとき
MCP・コネクタ

コスト構造の理解

サブスクとAPIは別の財布。重い処理は別枠で消費されることが多い。

  • 通常のチャット利用は、月額サブスクの範囲内で収まることが多い
  • 自動化（API経由・夜間の自動実行など）は、別途トークン単位の課金になりやすい
  • 「無料に見える機能」も、裏側で使っているモデルやツールによって負荷が変わる
今日から使えるアクション

① プロンプトに役割・条件・出力形式を入れる　② よく使う前提はProjects／カスタム指示に保存する　③ 「繋ぐ・調べる・実行する」を分けて考える
